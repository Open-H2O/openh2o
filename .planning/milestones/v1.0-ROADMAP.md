# Milestone v1.0: MVP

**Status:** SHIPPED 2026-05-24
**Phases:** 1-8
**Total Plans:** 22

## Overview

Stand up an AI-deployable water accounting platform from scratch. Docker infrastructure, 48-table domain model, spatial CRUD with maps, double-entry accounting engine, 8 external data adapters, state reporting, health monitoring, and complete deployment documentation.

## Milestone Summary

**Key Decisions:**
- Tailwind standalone binary (no Node.js anywhere in the stack)
- Split settings pattern (base/local/production) with django-environ
- ParcelLedger double-entry accounting (supply positive, usage negative)
- Staged import with duplicate detection for parcels; direct creation for wells
- BaseAdapter registry pattern for extensible data source integration
- OpenET 3-stage pipeline (trigger, poll, retrieve)
- Public health endpoint for monitoring tool access
- Phases 8+9 merged into single Phase 8 after scope review

**Issues Resolved:**
- Docker library path hardcoding (GDAL/GEOS found automatically on Debian)
- AUTH_USER_MODEL boot issue (minimal stub in core/models.py)
- django-allauth deprecation (ACCOUNT_LOGIN_METHODS/ACCOUNT_SIGNUP_FIELDS)
- CSS variable name mismatches (plan spec vs actual token names)
- Checkpoint automation failures (led to GSD framework reform)

**Issues Deferred:**
- ISSUE-001: RechargeSite missing zone FK
- ISSUE-002: WaterRight missing parcel FK
- OpenET API key not yet requested

**Technical Debt Incurred:**
- Template-logic conditional colors kept inline (irreducible for dynamic values)
- Some OKLCH colors hardcoded inline where no token existed (health badges)
- No automated test suite (deferred to post-MVP)
