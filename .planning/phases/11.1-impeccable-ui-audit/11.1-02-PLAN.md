<task type="auto">
  <name>Task 1: Apply fixes from critique and audit findings</name>
  <files>Determined by Plan 01 findings (likely: static/css/app.css, templates/*.html, possibly forms.py files)</files>
  <action>
Read the critique-findings.md and audit-findings.md reports from Plan 01. Read 11.1-01-SUMMARY.md for the decided fix scope (P0 only, P0+P1, or P0+P1+P2).

For each issue in scope, apply the fix. Common fix categories from /impeccable audits:

**Accessibility fixes (if flagged):**
- Add missing ARIA labels to interactive elements (buttons, links, form controls)
- Add `aria-label` or `aria-labelledby` to map container and map controls
- Ensure all form inputs have associated `<label>` elements (not just placeholder text)
- Add `role` attributes to landmark elements if missing
- Fix heading hierarchy (h1 → h2 → h3, no skipped levels)
- Add skip-to-content link in base.html if missing

**Theming fixes (if flagged):**
- Replace any hard-coded hex colors with design token CSS variables from tokens.css
- Ensure all text colors use token variables (--text-primary, --text-secondary, --text-tertiary)
- Fix any background colors not using surface tokens (--surface-base, --surface-card, --surface-elevated)

**Responsive fixes (if flagged):**
- Ensure touch targets are at least 44x44px (buttons, links, map controls)
- Fix any horizontal overflow issues found on specific pages
- Add missing breakpoint rules for pages not covered in Phase 11

**Anti-pattern fixes (if flagged):**
- Remove or replace any AI slop tells (gradient text, glassmorphism, dark glows, etc.)
- Fix visual hierarchy issues (unclear primary actions, competing focal points)
- Address cognitive load problems (too many options visible, missing progressive disclosure)

**Map-specific fixes (if flagged):**
- Add ARIA labels to map controls (zoom buttons, layer toggles)
- Ensure map popup content is keyboard-accessible
- Fix any touch target issues on map interactive elements

Group related fixes into logical commits (e.g., one commit for accessibility, one for theming, one for responsive). Do NOT create one commit per individual fix.

IMPORTANT: Follow the patterns established in Phase 11. Use existing CSS classes where possible. Add new utility classes to app.css only when no existing class fits. Do NOT introduce new CSS files or change the Tailwind config.
  </action>

- docker compose exec web python manage.py check --deploy passes
