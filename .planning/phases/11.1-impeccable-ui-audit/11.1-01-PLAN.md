**Assessment B (Automated Detection):** Runs the bundled detector (npx impeccable detect) against HTML/template source files, plus browser overlay visualization if available.

Target scope: Focus the critique on the three most important user flows:
1. Landing page → Parcels list → Parcel detail (the primary CRUD flow)

- Nielsen heuristic scores (10 scores, 0-4 each)
- AI slop verdict (pass/fail with specific tells)

- Priority issues (3-5 with what/why/fix)

Save the full critique output to `.planning/phases/11.1-impeccable-ui-audit/critique-findings.md`.

<task type="auto">
  <name>Task 2: Run /impeccable audit on OpenH2O UI</name>
  <files>N/A (skill invocation producing report output)</files>
  <action>
Invoke the /impeccable audit skill targeting the deployed OpenH2O UI. The audit checks 5 technical dimensions, each scored 0-4:

1. **Accessibility** - contrast ratios, ARIA labels, keyboard nav, semantic HTML, form labels
2. **Performance** - layout thrashing, expensive animations, lazy loading, bundle size
3. **Responsive Design** - fixed widths, touch targets (44x44px min), horizontal scroll, breakpoints
4. **Theming** - design token usage, hard-coded colors, dark mode consistency
5. **Anti-Patterns** - AI slop tells, design quality issues

Target scope: Same key pages as critique, plus:
- A form page (parcels/create or accounting/ledger/create) for form accessibility
- The health dashboard for data-heavy layout evaluation
- Login page (accounts/login) for first-impression assessment

The audit produces:
- 5-dimension scorecard (total out of 20)
- Rating band (Excellent/Good/Acceptable/Poor/Critical)
- Issues by severity (P0/P1/P2/P3)

Save the full audit output to `.planning/phases/11.1-impeccable-ui-audit/audit-findings.md`.

<task type="checkpoint:decision" gate="blocking">

  <context>
Phase 11 already did a UI quality sweep (responsive CSS, form classes, landing page, favicon, SVG icons). Phase 11.1 is a quality gate before documentation (Phase 12). Fixing too much risks scope creep; fixing too little means docs will screenshot a flawed UI.

After Tasks 1-2 complete, compile findings from both reports into a single prioritized list:
- **P0 (fix now):** Accessibility failures that block WCAG AA, broken interactions, AI slop tells
- **P1 (fix now):** Visual hierarchy problems, inconsistent theming, responsive breakage
- **P2 (defer to Phase 12+):** Nice-to-have polish, minor spacing, advanced a11y (AAA)
- **P3 (won't fix):** Intentional design choices, out-of-scope for this milestone

Present the compiled list with counts per priority level and estimated fix effort.
