**Admin best practices for this project:**
- list_display: show the most useful columns (5-7 max)
- search_fields: text fields users would search by
- list_filter: status fields, types, date fields
- For models with ForeignKeys: use raw_id_fields for large tables, autocomplete_fields where useful
- For spatial models: do NOT register with GeoModelAdmin (requires additional JS setup, defer to Phase 3)
- For ParcelLedger: list_display should show parcel, amount (with sign), water_type, effective_date
- For SiteConfig: use a singleton-aware admin that prevents adding a second instance
