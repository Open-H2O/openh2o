<task type="auto">
  <name>Task 2: Configure email backend and Google OAuth</name>
  <files>config/settings/base.py, config/settings/local.py, config/settings/production.py, .env.example</files>
  <action>
  In config/settings/base.py:
  - Add EMAIL_BACKEND = env("EMAIL_BACKEND", default="django.core.mail.backends.console.EmailBackend")
  - Add DEFAULT_FROM_EMAIL = env("DEFAULT_FROM_EMAIL", default="noreply@openh2o.com")
  - Add EMAIL_HOST = env("EMAIL_HOST", default="")
  - Add EMAIL_PORT = env.int("EMAIL_PORT", default=587)
  - Add EMAIL_USE_TLS = env.bool("EMAIL_USE_TLS", default=True)
  - Add EMAIL_HOST_USER = env("EMAIL_HOST_USER", default="")
  - Add EMAIL_HOST_PASSWORD = env("EMAIL_HOST_PASSWORD", default="")

  In config/settings/local.py:
  - Set EMAIL_BACKEND = "django.core.mail.backends.console.EmailBackend" (prints emails to console in development)

  In config/settings/production.py:
  - Set EMAIL_BACKEND = "django.core.mail.backends.smtp.EmailBackend" (real SMTP in production)

  In .env.example, add:
  - EMAIL_BACKEND, DEFAULT_FROM_EMAIL, EMAIL_HOST, EMAIL_PORT, EMAIL_USE_TLS, EMAIL_HOST_USER, EMAIL_HOST_PASSWORD
  - GOOGLE_OAUTH_CLIENT_ID (optional)
  - GOOGLE_OAUTH_CLIENT_SECRET (optional)

  Google OAuth setup:
  - In base.py, add a conditional block that reads GOOGLE_OAUTH_CLIENT_ID and GOOGLE_OAUTH_CLIENT_SECRET from env
  - Add SOCIALACCOUNT_PROVIDERS config for google with scope: ["profile", "email"]
  - The SiteConfig.allow_google_oauth flag controls whether the OAuth button appears in templates (template-level check, not settings-level)
  - In login.html, add conditional: {% if site_config.allow_google_oauth %}{% load socialaccount %}{% provider_login_url "google" %}{% endif %}
  - Create a context processor or template tag to make SiteConfig available in templates:
    core/context_processors.py with site_config(request) that returns {"site_config": SiteConfig.objects.first()}
  - Add "core.context_processors.site_config" to TEMPLATES context_processors in base.py
  </action>
  <verify>docker compose exec web python manage.py check succeeds. Console email backend works in dev (trigger password reset, see output in docker logs).</verify>
  <done>Email backend configured (console for dev, SMTP for prod). Google OAuth provider ready but disabled by default. SiteConfig context processor exposes toggle to templates.</done>
