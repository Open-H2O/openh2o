<task type="auto">
  <name>Task 1: Water budget dashboard</name>
  <files>accounting/views.py, accounting/urls.py, templates/accounting/dashboard.html, templates/accounting/partials/_dashboard_content.html, templates/partials/_sidebar.html</files>
  <action>
  **Dashboard view (accounting/views.py):**
  - dashboard: Main dashboard showing water budget overview. Accepts optional ?period=<pk> query param.
  - If no period selected, default to the most recent ReportingPeriod (by start_date).
  - Compute for the selected period:
    1. Account summaries: for each WaterAccount (active), call account_balance(account, period). Show: account name, supply, usage, net, allocation (from AllocationPlan sum for parcels in account's zones), remaining (allocation - usage abs).
    2. Zone summaries: for each Zone, call zone_balance(zone, period). Show: zone name, total supply, total usage, net, total allocation (AllocationPlan for this zone+period), remaining.
    3. Totals: grand total supply, usage, net across all accounts.
  - Period selector: dropdown of all reporting periods, HTMX-powered (hx-get reloads _dashboard_content.html partial when changed).

  **Dashboard template (templates/accounting/dashboard.html):**
  - Top row: 3 stat cards (Total Supply green, Total Usage red, Net Balance conditional color). Use large text, tabular-nums. Card-raised styling.
  - Account section: table with account rows. Columns: Account, Supply (AF), Usage (AF), Net (AF), Allocation (AF), Remaining (AF). Net and Remaining color-coded (green positive, furnace-orange negative). Link account name to account detail.
  - Zone section: similar table with zone rows.
  - If no reporting periods exist, show a "Create your first reporting period" call-to-action linking to period create.

  **URL:** Add path("dashboard/", dashboard, name="dashboard") to accounting/urls.py.
  **Sidebar:** Update the Dashboard link at top of sidebar (currently points to index) to point to accounting:dashboard. Keep the index view but make it redirect to dashboard if user is logged in (update config/views.py index view).

  **Design:** Use VanderDev tokens throughout. Stat cards use elevated surface with pop-shadow. Numbers use JetBrains Mono (font-family: var(--font-mono)). Supply = --color-forest-teal-500, Usage = --color-furnace-orange-500, positive net = --color-forest-teal-400, negative net = --color-furnace-orange-400.
  </action>

  <done>Dashboard renders with stat cards, account table, zone table. Period selector filters via HTMX. Color coding works. Empty state shows CTA.</done>
