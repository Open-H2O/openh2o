  <action>
  Wire URL routing:
  - Create urls.py for each app: parcels, wells, surface, recharge, geography
  - Include all in config/urls.py under appropriate prefixes: /parcels/, /wells/, /surface/, /recharge/, /map/
  - All views require login (@login_required decorator)

  Create GeoJSON API endpoints (one view per app that returns a FeatureCollection):
  - parcels/views.py: parcels_geojson -- serialize Parcel queryset (only records with geometry not null) using django.core.serializers.serialize('geojson', ..., geometry_field='geometry', fields=['parcel_number', 'owner_name', 'area_acres', 'status']). Return JsonResponse.
  - wells/views.py: wells_geojson -- serialize Well queryset using geometry_field='location', fields=['name', 'well_registration_id', 'status', 'depth_ft', 'capacity_gpm']. Return JsonResponse.
  - surface/views.py: pods_geojson -- serialize PointOfDiversion queryset using geometry_field='location', fields=['name', 'stream_name', 'max_rate_cfs', 'status']. Include water_right.right_id via annotation or manual serialization if needed.
  - recharge/views.py: recharge_sites_geojson -- serialize RechargeSite queryset using geometry_field='location', fields=['name', 'site_type', 'capacity_acre_feet', 'status'].

  URL patterns for GeoJSON: /parcels/geojson/, /wells/geojson/, /surface/pods/geojson/, /recharge/sites/geojson/

  Note: GeoDjango's serialize('geojson') returns a string, not a dict. Use HttpResponse with content_type='application/json', or json.loads() then JsonResponse. The simpler approach: HttpResponse(serialized_data, content_type='application/json').

  Also create placeholder list views (empty for now, just returning a template stub) so the nav links don't 404: parcels_list, wells_list, water_rights_list, recharge_sites_list.
