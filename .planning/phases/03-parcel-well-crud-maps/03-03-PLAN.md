<task type="auto">
  <name>Task 1: Parcel list and detail views with HTMX search/filter and inline editing</name>
  <files>parcels/views.py, parcels/urls.py, templates/parcels/list.html, templates/parcels/detail.html, templates/parcels/partials/_list_results.html, templates/parcels/partials/_field_edit.html</files>
  <action>
  Parcel list view (parcels_list):
  - GET /parcels/ renders full page with search bar and status filter dropdown
  - Search input: hx-get="/parcels/?q=..." hx-target="#results" hx-trigger="keyup changed delay:300ms"
  - Status filter: hx-get="/parcels/?status=active" hx-target="#results"
  - View checks request.headers.get('HX-Request'): if HTMX, return only the _list_results.html partial; if full page, return list.html wrapping the partial
  - Query: filter by parcel_number__icontains OR owner_name__icontains for search, status__exact for filter
  - Pagination: Django Paginator, 25 per page, HTMX loads next page
  - Table columns: Parcel Number (link to detail), Owner, Acres, Status, Zone(s)
  - Include count badge showing total matching records

  Parcel detail view (parcel_detail):
  - GET /parcels/<pk>/ renders detail page
  - Sections: Parcel Info (number, owner, area, status, address, notes), Zone Memberships, Related Wells (via WellIrrigatedParcel), Recent Ledger Entries (last 10)
  - Each editable field (owner_name, area_acres, status, address, notes) renders as text with a pencil icon
  - Click pencil: hx-get="/parcels/<pk>/edit-field/?field=owner_name" loads an inline form input
  - Submit: hx-patch="/parcels/<pk>/edit-field/" with field name and new value, returns the updated text span
  - parcels/views.py: parcel_edit_field view handles GET (return form partial) and PATCH (validate, save, return updated value partial)
  - Show a small embedded map of just this parcel's geometry if it exists (MapLibre with single GeoJSON feature, using the {% block map_scripts %} from base.html)

  Template styling: Use VanderDev card pattern (bg-card surface, border-default, rounded corners). Table uses hover highlight. Status badges use colored pills (active=green, inactive=gray, pending=gold).
  </action>
  <verify>docker compose up -d --build. Create a test parcel via admin. Visit /parcels/ -- see it in the list. Search by number. Click into detail. Click pencil on owner_name, edit, submit -- value updates without page reload.</verify>
  <done>Parcel list shows searchable/filterable table. Parcel detail shows all info with inline editing. HTMX partial responses work. Related wells and ledger entries display (empty if none linked).</done>
</task>
