  <name>Task 1: Add Chart.js CDN and build station detail telemetry chart</name>
  <files>templates/base.html, templates/datasync/station_detail.html, datasync/views.py, datasync/urls.py</files>
  <action>
  **Chart.js CDN:**
  Add Chart.js v4 via CDN to base.html in a new optional block (like `map_scripts`):
  ```html
  {% block chart_scripts %}{% endblock %}
  ```
  Station detail template will include:
  ```html
  {% block chart_scripts %}
  <script src="https://cdn.jsdelivr.net/npm/chart.js@4"></script>
  {% endblock %}
  ```
  This avoids loading Chart.js on every page. Only pages that need charts include it.

  **API endpoint for chart data:**
  Add a new view `station_chart_data(request, pk)` in `datasync/views.py` that returns JSON:
  ```json
  {
    "labels": ["2024-01-01", "2024-01-02", ...],
    "datasets": [
      {"label": "Reservoir Storage (AF)", "data": [1234.5, 1235.2, ...], "parameter_code": "15"}
    ],
    "parameters": [{"code": "15", "name": "Reservoir Storage", "unit": "AF"}, ...]
  }
  ```
  Query `DataRecordStaging` filtered by station, status='published', ordered by `observation_date`.
  Accept query params: `?parameter=15&days=30` (default: first parameter, last 30 days).
  Wire to URL: `path('<int:pk>/chart-data/', station_chart_data, name='station_chart_data')`.

  **Chart on station detail page:**
  Replace the "Recent Data Records" table card in the right column with a new chart card above it.
  The chart card contains:
  - A row of controls: parameter selector dropdown (populated from station.parameters) + date range buttons (7d / 30d / 90d / 1y)
  - A `<canvas>` element for Chart.js
  - The chart uses the VanderDev dark theme:
    - Background: transparent (card background shows through)
    - Grid lines: `var(--color-border)` equivalent hex `rgba(100,140,180,0.07)`
    - Tick labels: `#8899aa` (text-secondary)
    - Line color: determined by freshness (green `#22c55e` for fresh stations, amber `#f59e0b` for stale)
    - Point color: same as line
    - Fill: gradient from line color at 20% opacity to transparent
    - Font: Public Sans (Chart.js global `defaults.font.family`)

  **HTMX integration for chart updates:**
  When user clicks a parameter or date range button, use HTMX to fetch from the chart-data endpoint and re-render the chart via a small inline `<script>` that calls `chart.update()`. Or simpler: use vanilla JS fetch + Chart.js update since HTMX isn't ideal for canvas-based charts.

  **Keep the Recent Data Records table** below the chart, but limit it to the 10 most recent records (currently unlimited). Add a "View all" link if there are more.

  **Chart.js global config** (set once in the chart script block):
  ```javascript
  Chart.defaults.font.family = "'Public Sans', sans-serif";
  Chart.defaults.color = '#8899aa';
  Chart.defaults.borderColor = 'rgba(100,140,180,0.07)';
  ```
  </action>
  <verify>
  `grep 'chart.js' templates/base.html` confirms CDN block exists.
  `grep 'station_chart_data' datasync/urls.py` confirms endpoint wired.
  `grep 'canvas' templates/datasync/station_detail.html` confirms chart element exists.
  Read station_detail.html to verify parameter selector and date range buttons present.
  </verify>
  <done>Station detail page shows a Chart.js time-series chart with parameter dropdown and date range buttons. Chart uses VanderDev dark theme colors. Chart data served via JSON endpoint. Recent records table limited to 10 rows.</done>
