  <action>
  **ISSUE-001 fix:** Add an optional `zone` FK to `RechargeSite`:
  ```python
  zone = models.ForeignKey("geography.Zone", on_delete=models.SET_NULL, null=True, blank=True, related_name="recharge_sites")
  ```
  Run `python manage.py makemigrations recharge`.

  **ISSUE-002 fix:** Add a `WaterRightParcel` junction table to the `surface` app (not a direct FK, because one water right can apply to multiple parcels):
  ```python
  class WaterRightParcel(models.Model):
      water_right = models.ForeignKey(WaterRight, on_delete=models.CASCADE, related_name="water_right_parcels")
      parcel = models.ForeignKey("parcels.Parcel", on_delete=models.CASCADE, related_name="water_right_parcels")
      notes = models.TextField(blank=True)

      class Meta:
          unique_together = [("water_right", "parcel")]

      def __str__(self):
          return f"{self.water_right} â†’ {self.parcel}"
  ```
  Run `python manage.py makemigrations surface`.

  **Update accounting/services.py:**
  - `create_recharge_ledger_entries`: If `zone` param is None, fall back to `recharge_event.recharge_site.zone`. Raise ValueError if both are None.
  - `create_diversion_ledger_entry`: If `parcel` param is None, look up via `WaterRightParcel` through `diversion_record.point_of_diversion.water_right`. Use first linked parcel. Raise ValueError if no parcel found and none supplied.
  - Keep the explicit param path working for backward compatibility.

  **Update seed_demo_data.py:**
  - Set `zone` on each RechargeSite when creating them (they're already in zone-adjacent coordinates).
  - Create WaterRightParcel links between demo water rights and their holder's parcels.

  **Do NOT** change any templates or views -- the FK additions are model-layer only. The explicit param signature stays available.
