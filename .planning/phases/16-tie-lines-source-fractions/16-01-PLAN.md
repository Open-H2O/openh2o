<task type="auto">
  <name>Task 1: Create tie-lines GeoJSON endpoint</name>
  <files>geography/views.py, geography/urls.py</files>
  <action>
Add a new view `tie_lines_geojson(request)` in geography/views.py that:

1. Queries WellIrrigatedParcel with select_related('well', 'parcel') — these are groundwater (GW) tie lines
2. Queries PointOfDiversionParcel with select_related('point_of_diversion', 'parcel') — these are surface water (SW) tie lines
3. For each link, computes parcel centroid using `parcel.geometry.centroid` (PostGIS ST_Centroid)
4. Returns a GeoJSON FeatureCollection of LineString features where each feature has:
   - geometry: LineString from source point (well.location or pod.location) to parcel centroid
   - properties: { source_type: "gw"|"sw", source_name, parcel_number, fraction, source_id, parcel_id }

Skip any link where parcel.geometry is NULL (no centroid computable).

Use `@login_required` decorator matching existing endpoint pattern.
Build GeoJSON manually (not django serializer — these are computed geometries, not model fields).
Return as HttpResponse with content_type="application/json".

Register URL: `path("tie-lines/geojson/", views.tie_lines_geojson, name="tie_lines_geojson")` in geography/urls.py.
  </action>
  <verify>
docker compose exec web python manage.py shell -c "from geography.views import tie_lines_geojson; print('import ok')"
curl -s http://localhost/geography/tie-lines/geojson/ (after auth) returns valid GeoJSON with features
  </verify>
  <done>Endpoint returns FeatureCollection with LineString features for all well→parcel and POD→parcel links. Each feature has source_type, fraction, and name properties.</done>
</task>

<task type="auto">
  <name>Task 3: Apply fraction weighting in reporting exports</name>
  <files>reporting/generators.py</files>
  <action>
Modify the GEARS CSV generator (generate_gears_csv) to weight extraction volumes by WellIrrigatedParcel.fraction:

1. In the "by_well" method: When iterating WellIrrigatedParcel links, store the fraction alongside the well. When accumulating volume for a key, multiply by fraction: `rows[key]["volume"] += abs(entry.amount_acre_feet) * wip.fraction`. This correctly distributes a parcel's total extraction across its source wells proportionally.

2. In the CalWATRS A1 generator: Add a "Source Fraction" column after the well/POD identifier. For each WellIrrigatedParcel or PointOfDiversionParcel link, include the fraction as a decimal (0.0-1.0). When a parcel has multiple sources (combined use), each source gets its own row with its fraction, and volumes are multiplied by fraction.

3. Add a "Combined Use" indicator column to CalWATRS A1: if a parcel appears in BOTH WellIrrigatedParcel and PointOfDiversionParcel, mark as "Combined" (otherwise "GW Only" or "SW Only").

Do NOT change the "by_et" method — ET-based reporting is parcel-level and doesn't need source attribution.

Preserve backward compatibility: if all fractions are 1.0 (single-source parcels), output is identical to current behavior.
