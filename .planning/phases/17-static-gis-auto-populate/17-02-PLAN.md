**API endpoint verified (2026-05-25):**
- DWR LightBox Parcels MapServer: `https://gis.water.ca.gov/arcgis/rest/services/Planning/i15_Parcels_Assessor_Lightbox/MapServer/0`
- Key fields: PARCEL_APN, COUNTYNAME, SITE_ADDR, SITE_CITY, SITE_STATE, SITE_ZIP
- No owner info available (stripped by DWR)
- Geometry: esriGeometryPolygon, native EPSG:3857 (request outSR=4326)
- Max record count: 1,500 per query page
- Pagination: resultOffset/resultRecordCount (same pattern as B118)
- Spatial queries: esriSpatialRelIntersects supported
- No auth required

- Step registry pattern: add new step method, wire into OrderedDict
- Idempotency via existence check before create

1. Add module-level constant:
   ```python
   LIGHTBOX_PARCELS_URL = (
       "https://gis.water.ca.gov/arcgis/rest/services/Planning/"
       "i15_Parcels_Assessor_Lightbox/MapServer/0/query"
   )
   ```

2. Implement `_step_parcels(self, boundary, dry_run)`:
   - Import `query_feature_server` and `geos_to_esri_geometry` from geography.services.arcgis
   - Convert boundary geometry to ArcGIS JSON for spatial query
   - Use `query_feature_server` as a generator (process page-by-page for efficiency)
   - Request only needed fields: `PARCEL_APN,SITE_ADDR,SITE_CITY,SITE_STATE,SITE_ZIP`
   - Request outSR=4326 (service is natively 3857)
   - For each page of features:
     a. Collect APNs from the page
     b. Bulk-check which APNs already exist: `Parcel.objects.filter(parcel_number__in=apns).values_list('parcel_number', flat=True)`
     c. For new parcels only:
        - Convert esri polygon geometry to GEOSGeometry using `esri_polygon_to_geos()`
        - Build address string from SITE_ADDR + SITE_CITY + SITE_STATE + SITE_ZIP
        - Create Parcel object (parcel_number, geometry, address, status="active")
     d. Use `Parcel.objects.bulk_create(new_parcels, ignore_conflicts=True)` for efficiency
     e. Log progress: page number, parcels created this page, running total
   - In dry_run mode: count and log what would be created, don't write
   - Handle edge cases:
     - Skip features with empty/None PARCEL_APN
     - Skip features with no geometry (warn)
     - Handle API errors gracefully (log and return partial count)
   - Return total count of parcels created

3. Import Parcel model at the top of the file:
   `from parcels.models import Parcel`
