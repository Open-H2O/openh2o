<task type="auto">
  <name>Task 1: Create ArcGIS REST API client utility</name>
  <files>geography/services/__init__.py, geography/services/arcgis.py</files>
  <action>
Create `geography/services/` package with `__init__.py` and `arcgis.py`.

The `arcgis.py` module provides a reusable client for querying ArcGIS REST FeatureServer endpoints. It will be used by multiple auto_populate steps (B118 basins in this plan, LightBox parcels in 17-02).

Implement these functions:

1. `query_feature_server(url, where="1=1", geometry=None, geometry_type=None, spatial_rel=None, out_fields="*", return_geometry=True, out_sr=4326, max_record_count=1000)` — Generator function that:
   - Builds query params: where, geometry (as JSON envelope or rings), geometryType, spatialRel, outFields, returnGeometry, outSR, f=json, resultRecordCount, resultOffset
   - Yields pages of features (list of dicts with 'attributes' and 'geometry' keys)
   - Handles pagination: checks `exceededTransferLimit` in response, increments resultOffset
   - Rate limits: 0.5s sleep between pages
   - Retries: up to 3 retries with exponential backoff on HTTP errors
   - Logs page count and record count

2. `query_by_boundary(url, boundary_geometry, out_fields="*", return_geometry=True)` — Convenience wrapper that:
   - Takes a GEOSGeometry (MultiPolygon) boundary
   - Converts boundary to ArcGIS JSON rings format using `boundary_geometry.json` (GeoJSON) then transforms to esri rings
   - Calls query_feature_server with geometry, geometryType=esriGeometryPolygon, spatialRel=esriSpatialRelIntersects
   - Returns flat list of all features (consumes the generator)

3. `esri_polygon_to_geos(esri_geometry)` — Converts ArcGIS JSON polygon geometry (with 'rings' key) to django.contrib.gis.geos MultiPolygon (SRID 4326). Handles single-ring and multi-ring polygons.

4. `geos_to_esri_geometry(geos_geometry)` — Converts GEOSGeometry to ArcGIS JSON geometry dict for use in spatial queries. Uses the geometry's GeoJSON coordinates and restructures to esri rings format.

Use `requests` library (already in project dependencies). Use `logging.getLogger(__name__)` for all logging.
  </action>
