  <files>
    datasync/views.py,

    templates/datasync/station_list.html,

    templates/datasync/station_add.html
  </files>
  <action>
    **datasync/urls.py:** Create URL configuration with app_name = 'datasync'. Routes:
    - stations/ → station_list
    - stations/add/ → station_add
    - stations/&lt;int:pk&gt;/ → station_detail
    - stations/&lt;int:pk&gt;/toggle/ → station_toggle (HTMX POST)
    - stations/geojson/ → stations_geojson

    **config/urls.py:** Add `path("datasync/", include("datasync.urls"))` to urlpatterns.

    **views.py:**
    - `station_list`: Paginated list of MonitoredStation with HTMX search (station_name, external_station_id) and filter by data_source and is_active. Follow the parcels/list.html pattern.
    - `station_detail`: Show station info, data source, parameters, last_data_at, recent DataRecordStaging entries (last 20), recent DataSyncLog entries. Include a small MapLibre map showing just this station's location.
    - `station_toggle`: HTMX POST endpoint to toggle is_active. Returns updated toggle partial.
    - `station_add`: Form to add custom station (select data_source, enter external_station_id, station_name, lat/lon). Creates MonitoredStation with is_active=True.
    - `stations_geojson`: GeoJSON FeatureCollection of all active MonitoredStation records. Include properties: pk, station_name, external_station_id, data_source_code, is_active, last_data_at.

    **Templates:** Follow established dark-mode styling with VanderDev tokens. Use the same card/table patterns from parcels and accounting pages. Station list shows source badge (colored by data_source.code), station name, external ID, active toggle, last data date. Detail page shows station metadata + map + recent data table.

    **Sidebar:** Add "Stations" link under the Data section in templates/partials/_sidebar.html.
  </action>
