  <action>
    Create `datasync/adapters/` package.

    **base.py — BaseAdapter abstract class:**
    - Abstract methods: `fetch(station, start_date, end_date)`, `parse(raw_data)`, `validate(records)`, `discover_stations(boundary_geometry, radius_km)`
    - Concrete methods: `sync(station, start_date, end_date)` (orchestrates fetch→parse→validate→stage→publish), `stage(records, sync_log)` (bulk creates DataRecordStaging), `publish(sync_log)` (promotes staged→published, updates station.last_data_at)
    - `mock` property: if `settings.DATASYNC_MOCK_MODE` is True or DataSource.is_active is False, call `fetch_mock()` instead of `fetch()`
    - `fetch_mock(station, start_date, end_date)` — returns fixture data from `datasync/fixtures/{source_code}.json`
    - Rate limiting: `rate_limit_seconds` class attribute (default 1.0), enforced between API calls via `time.sleep()` in `fetch()`
    - Retry: `max_retries` class attribute (default 3), exponential backoff on HTTPError/ConnectionError
    - All methods log to DataSyncLog (create on start, update on completion/failure)
    - Use `requests` library for HTTP (already in project via django dependencies). Do NOT add httpx or aiohttp — management commands are synchronous.

    **__init__.py — Adapter registry:**
    - `ADAPTER_REGISTRY` dict mapping DataSource.code → adapter class
    - `get_adapter(source_code)` function that returns instantiated adapter
    - Import all concrete adapters to auto-populate registry

    **Management commands:**
    - `sync_source`: `manage.py sync_source <code> [--start YYYY-MM-DD] [--end YYYY-MM-DD] [--mock]`. Fetches data for all active stations of that source. Creates DataSyncLog. Prints summary.
    - `sync_all`: `manage.py sync_all [--mock]`. Iterates active DataSources, calls sync_source for each. Prints per-source summary.
    - `discover_stations`: `manage.py discover_stations <code> [--radius 50] [--boundary-name "My GSA"]`. Uses Boundary geometry + radius to find nearby stations. Creates MonitoredStation records (is_active=False by default). Prints discovery count.
    - `check_source_health`: `manage.py check_source_health [<code>]`. For each source: checks last_sync_at freshness, counts active stations, reports any recent failures. Prints health table.
  </action>

  <action>
    Each adapter extends BaseAdapter. Each has `fetch()`, `parse()`, `validate()`, `discover_stations()` implemented. Each has a matching JSON fixture in `datasync/fixtures/` for mock mode.

    **CDEC (cdec.py):**
    - Base URL: https://cdec.water.ca.gov/dynamicapp/req/JSONDataServlet
    - Params: Stations, SensorNums, dur_code, Start, End
    - No auth required
    - Parameters: reservoir storage (15), river stage (1), flow (20), precipitation (2)
    - discover_stations: CDEC station list CSV, filter by lat/lon within boundary

    **USGS (usgs.py):**
    - Base URL: https://waterservices.usgs.gov/nwis/iv/ (instantaneous values)
    - Also supports: /dv/ (daily values)
    - Params: sites, parameterCd, startDT, endDT, format=json
    - No auth required (public API, but be polite with rate limiting)
    - Parameters: discharge (00060), gage height (00065), water temperature (00010)
    - discover_stations: bBox parameter with boundary extent

    **OpenET (openet.py):**
    - 3-stage async pipeline:
      1. POST /raster/timeseries/submit — trigger job, get request_id
      2. GET /raster/timeseries/status/{request_id} — poll until "complete"
      3. GET /raster/timeseries/results/{request_id} — retrieve data
    - Auth: API key in header (X-API-KEY)
    - Parameters: ET (evapotranspiration) by field/parcel geometry
    - Note: OpenET works on field geometries, not station points — adapter receives parcel geometries as input
    - discover_stations: N/A for OpenET (geometry-based, not station-based) — return empty list

    **CIMIS (cimis.py):**
    - Base URL: https://et.water.ca.gov/api/data
    - Params: appKey, targets (station IDs), startDate, endDate, dataItems
    - Auth: AppKey in query param
    - Parameters: ETo (reference ET), precipitation, solar radiation, wind speed, temperature
    - discover_stations: CIMIS station info endpoint, filter by lat/lon

    **CNRFC (cnrfc.py):**
    - File-based: downloads forecast CSV files from https://www.cnrfc.noaa.gov/data/
    - No auth required
    - Parameters: streamflow forecast, precipitation forecast
    - discover_stations: parse station list page, filter by lat/lon

    **DWR WDL (dwr_wdl.py):**
    - Base URL: https://wdl.water.ca.gov/waterdatalibrary/
    - Params: station ID, date range
    - No auth required
    - Parameters: groundwater levels (depth to water, water surface elevation)
    - discover_stations: DWR groundwater well search API, filter by boundary bbox

    **DWR SGMA (dwr_sgma.py):**
    - Base URL: https://sgma.water.ca.gov/webservice/
    - REST endpoints for GSP data, monitoring wells, water levels
    - No auth required
    - Parameters: groundwater levels, subsidence, interconnected surface water
    - discover_stations: SGMA monitoring well endpoint, filter by basin/boundary

    **NOAA (noaa.py):**
    - Base URL: https://www.ncei.noaa.gov/cdo-web/api/v2/
    - Auth: Token in header
    - Params: datasetid, stationid, startdate, enddate, datatypeid
    - Parameters: PRCP (precipitation), TMAX, TMIN, SNOW
    - discover_stations: /stations endpoint with extent filter

    **Mock fixtures:** Each JSON file contains 10-20 sample records matching the adapter's parse() output format. Include realistic California water data values. Structure: `{"stations": [...], "records": [{"station_id": "...", "observation_date": "...", "parameter_code": "...", "value": ..., "unit": "..."}]}`.

    **Important:** All adapters must handle connection errors gracefully (log to DataSyncLog with status="failed", don't crash). OpenET adapter's poll loop should have a max_poll_attempts (default 30) with 10-second sleep between polls.
