<task type="auto">
  <name>Task 2: Infrastructure app with views and URL routing</name>
  <files>infrastructure/__init__.py, infrastructure/apps.py, infrastructure/views.py, infrastructure/urls.py, config/settings/base.py, config/urls.py</files>
  <action>
  Create a new Django app called `infrastructure` (views and templates only, no models). This app provides the unified "Add Infrastructure" form that creates records in existing apps (wells, surface, recharge).

  **App setup:**
  - `apps.py`: InfrastructureConfig with verbose_name "Infrastructure"
  - Register in INSTALLED_APPS in config/settings/base.py
  - Include URLs at `/infrastructure/` in config/urls.py

  **Views (infrastructure/views.py):**

  1. `infrastructure_list(request)` — GET: List all infrastructure across types. Query Well.objects.all(), PointOfDiversion.objects.filter(water_right__isnull=True) (standalone only), RechargeSite.objects.all(). Return combined list sorted by created_at desc. Template: infrastructure/list.html.

  2. `infrastructure_add(request)` — GET: Render the add form with type selector and map. POST: Parse form data, create the appropriate model instance based on `infra_type` field:
     - `well`: Create Well with name, location (PointField from submitted lat/lng), depth_ft, capacity_gpm, status, owner_name, notes
     - `diversion`: Create PointOfDiversion with name, location, stream_name, max_rate_cfs, status, notes (water_right=None)
     - `recharge_site`: Create RechargeSite with name, location (centroid), geometry (MultiPolygon if polygon drawn), site_type, capacity_acre_feet, status, operator, notes
     - `storage`: Create RechargeSite with name, location, geometry, site_type (storage_pond or storage_tank), capacity_acre_feet, status, notes

     After creating the main record, if `parcel_id` is submitted, create the appropriate junction record:
     - Well → WellIrrigatedParcel(well=well, parcel=parcel)
     - PointOfDiversion → PointOfDiversionParcel(point_of_diversion=pod, parcel=parcel)
     - RechargeSite → set zone FK if parcel is in a zone (no junction table exists)

     On success, redirect to the detail page of the created record (wells:detail, surface:water_right_detail, recharge:detail).

  3. `infrastructure_upload(request)` — POST (HTMX endpoint): Accept a file upload, parse it server-side, return GeoJSON for map preview. Handle three formats:
     - `.geojson` / `.json`: json.loads(), extract features
     - `.zip` (zipped Shapefile): Save to temp dir, open with django.contrib.gis.gdal.DataSource, extract features as GeoJSON
     - `.kml`: Save to temp dir, open with GDAL DataSource, extract features as GeoJSON
     Return JSON response with features array (each feature has geometry + properties).

  4. `infrastructure_geojson(request)` — GET: Return all infrastructure as a single GeoJSON FeatureCollection (wells as points, PODs as points, recharge sites as polygons with point fallback). Include type and name in properties.

  **URL patterns (infrastructure/urls.py):**
  ```
  app_name = "infrastructure"
  urlpatterns = [
      path("", infrastructure_list, name="list"),
      path("add/", infrastructure_add, name="add"),
      path("upload/", infrastructure_upload, name="upload"),
      path("geojson/", infrastructure_geojson, name="geojson"),
  ]
  ```

  Use @login_required on all views. Use GeoDjango GEOSGeometry for geometry parsing. For the upload view, use tempfile for Shapefile/KML extraction and clean up after.
  </action>

  **Form structure:**
  1. **Type selector** at top: 4 radio cards (Groundwater Well, Surface Water Diversion, Storage Pond/Tank, Recharge Site). Each card has an icon, name, and one-line description. Selecting a type shows/hides relevant fields via HTMX or vanilla JS (prefer JS to avoid round-trips).

  2. **Common fields** (all types): Name (text), Status (select: active/inactive/proposed), Notes (textarea)

  3. **Type-specific fields** (shown/hidden by JS):
     - Well: Depth (ft), Capacity (gpm), Owner Name, Well Registration ID
     - Diversion: Stream Name, Max Rate (cfs)
     - Storage: Storage Type (pond/tank select), Capacity (acre-feet)
     - Recharge Site: Site Type (spreading_basin/injection_well/streambed/asr_well select), Capacity (acre-feet), Operator

  4. **Geometry capture section** with two tabs presented with equal visual weight:
     - **"Plot on Map" tab**: MapLibre map with dark-matter basemap. Two draw modes:
       a) **Point mode** (default for well/diversion): Click map to place a marker. Only one marker at a time (clicking again moves it). Show lat/lng below the map in a read-only field. Store in hidden inputs `geometry_type=Point` and `geometry_json={"type":"Point","coordinates":[lng,lat]}`.
       b) **Polygon mode** (default for recharge/storage): Click to add vertices, shown as connected lines. Double-click or "Finish Drawing" button to close the polygon. Show vertex count and approximate area. Store in hidden inputs `geometry_type=Polygon` and `geometry_json=...`. Add "Clear" button to reset and start over.
       The mode switches automatically based on type selection (well/diversion = point, storage/recharge = polygon) but user can override.

     - **"Upload File" tab**: Drag-and-drop zone (reuse the upload-zone CSS class from wizard.html). Accept .geojson, .json, .zip (shapefile), .kml. On file select, POST to infrastructure:upload via HTMX. Response renders features on the map and populates the hidden geometry inputs. If multiple features in the file, show a feature picker.

  5. **Parcel linkage section** (collapsible, below geometry):
     - **Option A**: "Select existing parcel" dropdown (HTMX search of parcels by number/owner)
     - **Option B**: "Click parcel on map" toggle that loads the parcel GeoJSON layer on the map. Clicking a parcel polygon selects it and populates the parcel_id hidden input. Show selected parcel number.
     - **Option C**: "Draw new parcel" button that switches map to polygon draw mode for a parcel. On completion, show "Save as new parcel" mini-form (parcel number + owner name). Creates parcel via HTMX POST before main form submit.
     Visual indicator showing which parcel is linked (or "No parcel linked" in tertiary text).

  6. **Submit section**: "Create [Type Name]" button with type name updating dynamically. Cancel link back to infrastructure:list.

  **Map implementation details (NO external draw library):**
  - Initialize MapLibre with dark-matter-gl-style, zoom 8, center on boundary centroid if one exists in the system
  - Point mode: `map.on('click', handler)` adds/moves a GeoJSON source+circle layer. Use a draggable marker via maplibregl.Marker with `draggable: true`.
  - Polygon mode: Maintain a `vertices` array. Each click pushes [lng, lat]. Render as a GeoJSON LineString (while drawing) that becomes a Polygon (when finished). Use a dotted line while drawing, solid fill+stroke when complete.
  - File upload preview: Add uploaded features as a separate GeoJSON source with distinct styling (dashed outline).
  - Parcel layer: Load from parcels:geojson endpoint. Semi-transparent fill, click handler to select.
  - Coordinate display: Show EPSG:4326 coords with 6 decimal places below the map.

  - `.map-form-layout` two-column grid (1fr 1fr, collapse to single column at 768px)
  - `.draw-toolbar` for map mode buttons (Point/Polygon/Clear/Finish)
  - `.tab-pair` for Plot/Upload tab switcher
  - `.parcel-link-section` collapsible with smooth transition

  Use VanderDev tokens throughout: card-raised for containers, color-gold for active states, surface colors for form inputs.

  <name>Task 4: File upload parsing + parcel linkage endpoints</name>
  <files>infrastructure/views.py, infrastructure/urls.py</files>
  <action>
  Flesh out the infrastructure_upload view and add parcel-related HTMX endpoints.

  **File upload (infrastructure_upload):**
  The view skeleton was created in Task 2. Now implement the parsing logic:

  1. **GeoJSON (.geojson, .json):**
     - `json.loads(uploaded.read().decode('utf-8'))`
     - Handle Feature, FeatureCollection, raw geometry
     - Extract each feature's geometry and properties
     - Return JSON: `{"features": [{"geometry": {...}, "properties": {...}}, ...]}`

  2. **Zipped Shapefile (.zip):**
     - Save to `tempfile.mkdtemp()`
     - Extract zip contents
     - Find .shp file in extracted contents
     - Open with `django.contrib.gis.gdal.DataSource(shp_path)`
     - Iterate layer features, serialize geometry as GeoJSON, collect properties
     - Clean up temp dir in finally block
     - Return same JSON format

  3. **KML (.kml):**
     - Save uploaded file to temp path
     - Open with `DataSource(kml_path)` (GDAL supports KML natively)
     - Same iteration pattern as Shapefile
     - Clean up temp file

  For all formats:
  - Transform to SRID 4326 if source has different SRID
  - Limit to 500 features max (return error if exceeded)
  - Return HTTP 400 with error message for parse failures
  - Set Content-Type: application/json

  **Parcel search endpoint:**
  Add `parcel_search(request)` view:
  - GET with `?q=` parameter
  - Query Parcel.objects.filter(Q(parcel_number__icontains=q) | Q(owner_name__icontains=q))[:20]
  - Return HTMX partial with parcel options as clickable list items
  - Each item shows parcel_number, owner_name, area
  - Clicking sets parcel_id hidden input

  Add `parcel_create_inline(request)` view:
  - POST: Accept parcel_number, owner_name, geometry_json
  - Create Parcel with geometry from GeoJSON
  - Return HTMX partial showing the newly created parcel as selected

  **URL additions:**
  ```
  path("parcels/search/", parcel_search, name="parcel_search"),
  path("parcels/create/", parcel_create_inline, name="parcel_create"),
  ```

  Templates for parcel HTMX responses:
  - `templates/infrastructure/partials/_parcel_results.html` - search results list
  - `templates/infrastructure/partials/_parcel_selected.html` - selected parcel display
  </action>
