# Phase 19.2: Visual Overhaul & UX Refinement - Context

**Gathered:** 2026-05-25
**Status:** Ready for planning

<vision>
## How This Should Work

The platform should look and feel like a cross between vanderdev.net and a modern SaaS dashboard (Linear, Vercel). Professional, sharp, minimal. When a water district board member or GSA manager opens this for the first time, they should think "this is serious software" -- not a side project.

The current font is too playful. Replace it with a clean, professional typeface that matches the VanderDev aesthetic. Cards and tiles should be more square and modular, less rounded and casual. The overall layout should feel like siblings with vanderdev.net.

The California gold accent is a keeper, but it clashes slightly with the blue in the water drop logo. The full palette needs to be rebuilt to harmonize with the logo from the About page. Present 2-3 palette directions for comparison before committing.

The water drop logo in the top-left corner needs to be larger. It's the site anchor and should command presence.

"Water Rights" tab is renamed to "Surface Water." Users may or may not be rights-holders, and "Water Rights" is confusing. "Surface Water" matches the actual data: points of diversion, streams, flow rates.

Reports tab currently dumps users into a list. Instead, it should show both report types (GEARS and CalWATRS) as tabs on the same page. Users can see what each requires and switch between them without a decision gate.

The user wants to see mockups before building. Present palette options and layout concepts for approval before writing code.

</vision>

<essential>
## What Must Be Nailed

- **Professional first impression** -- font, spacing, palette, logo all working together so it reads as "serious software built for government water agencies"
- **Navigation clarity** -- every tab name makes sense to a water district manager without training. "Surface Water" not "Water Rights." Reports flow is obvious with GEARS/CalWATRS tabs.
- Both are equally critical. Can't ship visual polish without naming/flow cleanup or vice versa.

</essential>

<boundaries>
## What's Out of Scope

- No hard boundaries set. Use judgment during planning to keep the phase focused on visual + naming + flow refinement without feature creep.
- This is design + UX refinement, not new functionality. No new data models or business logic unless required to support the report tab restructure.

</boundaries>

<specifics>
## Specific Ideas

- **Font**: Match VanderDev. Current Public Sans feels too casual. Reference vanderdev.net's typography.
- **Cards/tiles**: More square, modular. Less rounded corners, more structured grid feel.
- **Palette**: Extract from water drop logo. Present 2-3 options side by side. Gold accent stays but may need shade adjustment. Fix gold-vs-blue clash.
- **Logo**: Larger in sidebar top-left. Anchor the brand.
- **"Water Rights" -> "Surface Water"**: Rename tab + any associated labels/breadcrumbs.
- **Reports page**: Two tabs (GEARS, CalWATRS) on one page. Each tab shows what the report contains, which reporting period, and submit/export options.
- **Reference material**: VanderDev site, plus existing GSD/impeccable-audit files in the repo for design patterns.
- **Mockups first**: User wants to see and approve visual direction before implementation.

</specifics>

<notes>
## Additional Context

The platform targets California GSAs (Groundwater Sustainability Agencies) and water districts. Many are small, underfunded agencies. The visual presentation matters because it builds trust that the software is legitimate and production-grade. A polished UI signals "this can go in front of a board" vs "this is a prototype."

VanderDev (vanderdev.net) is the parent design system. The two sites should feel like they belong to the same family without being identical.

</notes>
