# VanderDev reference (sibling design system):
# ~/Documents/GitHub/vanderoffice/VanderDev-Website/src/styles/tokens.css

**Tech stack available:** Django templates, HTMX, Tailwind standalone, CSS custom properties (OKLCH)
**Established patterns:** content-section-label, stat-card-accent-*, card-raised, sidebar-link active states, search-input-icon

**Constraining decisions:**
- Phase 12.1: oklch color space for surface tokens (matches VanderDev measurements)
- Phase 12.1: CSS background-image SVG pattern for icons (avoids template changes)
- Phase 15: Water drop logo generated (Contour Basin v2), stored at static/img/logo.png and favicon variants

**Critical finding:** Both OpenH2O and VanderDev already use Public Sans. The "too casual" perception stems from:
1. Border-radius too large (16px on cards vs VanderDev's 16px nominal but tighter visual density)
2. Neumorphic dual-source shadows (OpenH2O) vs clean single-source directional shadows (VanderDev Pacific Golden)
3. Missing font weight hierarchy (OpenH2O loads 400/600/800; VanderDev loads 300-800)
4. Gold accent (#E4A317) clashing with logo blues

The fix is NOT a font swap. It's: tighter radii, aligned shadows, expanded weight range, and harmonized palette.
</context>
