  <action>
Create `crontab.txt` in the project root with three cron entries. Each entry must use `cd /opt/openh2o && docker compose exec -T web python manage.py <command>` pattern (the -T flag is critical for non-interactive cron execution). Redirect stdout to /dev/null but capture stderr to a log file.

Entries:
- `sync_all --mock` daily at 2:00 AM (mock mode until OpenET API key is acquired; the --mock flag returns synthetic data so the sync pipeline exercises without hitting real APIs). Add a comment noting to remove --mock once API keys are configured.
- `run_health_checks` every 6 hours (0 */6 * * *)
- `prune_old_data --confirm` on the 1st of each month at 3:00 AM (the --confirm flag is required because the command defaults to dry-run)

Add header comment block explaining what each job does and how to customize paths.

Add Makefile targets:
- `install-cron`: Installs crontab.txt entries by appending to the current user's crontab (use `(crontab -l 2>/dev/null; cat crontab.txt) | crontab -` pattern to preserve existing entries). Print confirmation message.
- `show-cron`: Display current crontab entries (crontab -l).
- `sync`: Run sync_all manually (shortcut for `docker compose exec web python manage.py sync_all`).

Do NOT use `crontab crontab.txt` (that would overwrite existing entries). The append pattern is safer.
