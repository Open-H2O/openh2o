       - Add a build step that downloads the Tailwind standalone binary (linux-x64)
       - URL pattern: https://github.com/tailwindlabs/tailwindcss/releases/latest/download/tailwindcss-linux-x64
       - Run the binary to generate output.css during build
       - Do NOT install Node.js

    3. Create `tailwind.config.js` (yes, standalone reads this):
       - content: ['./templates/**/*.html']
       - theme.extend with VanderDev design tokens mapped to Tailwind (colors, fontFamily, borderRadius, spacing)

    **VanderDev design tokens (port from DESIGN.md + DESIGN.json):**

    4. Create `static/css/tokens.css` with CSS custom properties:
       - Colors from DESIGN.md: --color-base (#040608), --color-card (#080b10), --color-elevated (#0e1219), --color-hover (#141a22), --color-gold (#E4A317), --color-gold-hover (#D4952A), --color-blue (#1B7FAF), --color-text-primary (#e8edf4), --color-text-secondary (#8899aa), --color-text-tertiary (#4d5e6f), --color-border (rgba(100,140,180,0.07))
       - OKLCH ramps from DESIGN.json for furnace-orange, reservoir-blue, forest-teal (use for data viz and status colors)
       - Typography: --font-display (Public Sans), --font-mono (JetBrains Mono)
       - Spacing: --space-xs through --space-3xl
       - Shadows: --shadow-pop-sm, --shadow-pop-lg, --shadow-inset
       - Border radius: --radius-sm (6px), --radius-md (10px), --radius-lg (16px)

    5. Create `static/css/input.css`:
       - @tailwind base/components/utilities directives
       - @import './tokens.css'
       - Base styles: body background --color-base, text --color-text-primary, font-family Public Sans
       - Utility classes for cards (.card-raised), buttons (.btn-primary, .btn-secondary)

    **Base template:**

    6. Create `templates/base.html`:
       - HTML5 boilerplate, lang="en"
       - Google Fonts link: Public Sans (weights 400,600,800) + JetBrains Mono (400)
       - Link to static/css/output.css (Tailwind compiled)
       - HTMX via CDN: unpkg.com/htmx.org (latest stable)
       - django-htmx meta tag for CSRF: hx-headers='{"X-CSRFToken": "{{ csrf_token }}"}'
       - {% block content %} for page content
       - {% block title %} for page title
       - Minimal page shell: dark background, centered content area

    7. Create `templates/index.html`:
       - Extends base.html
       - Simple landing/status page showing: project name "Open Water Accounting Platform", version placeholder, "Stack is running" message
       - Use VanderDev design tokens: dark background, gold accent, card-raised panel, Public Sans text
       - This proves the full pipeline works: Django → template → Tailwind → design tokens → Caddy

    8. Create `config/views.py`:
       - Simple index view (TemplateView or function view) rendering index.html
       - Wire up in config/urls.py at root path

    **What to avoid and WHY:**
    - Do NOT use npm/npx for Tailwind — standalone binary only (no Node.js constraint)
    - Do NOT use Tailwind CDN play mode — it's development-only and bloats pages. Standalone binary compiles to production CSS.
    - Do NOT inline the entire DESIGN.json — pick the tokens that matter for Django templates. The full OKLCH ramps are for data viz colors; base UI needs ~20 custom properties.
    - Do NOT use @import in production CSS — Tailwind processes input.css into a single output.css.
    - Do NOT forget to add 'whitenoise.middleware.WhiteNoiseMiddleware' to middleware for static file serving in production.
  </action>
