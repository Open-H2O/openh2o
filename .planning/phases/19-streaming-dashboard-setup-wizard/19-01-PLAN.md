<task type="auto">
  <name>Task 3: Create setup wizard app with multi-step views</name>
  <files>setup/__init__.py, setup/apps.py, setup/views.py, setup/urls.py, config/settings/base.py, config/urls.py</files>
  <action>
Create a new Django app called "setup" (minimal: __init__.py, apps.py, views.py, urls.py — no models needed).

Register in INSTALLED_APPS (config/settings/base.py).

Views (all @login_required):

1. `setup_wizard(request)` — Step 1: Boundary selection
   - GET: Show form with two options: select existing Boundary from dropdown OR upload GeoJSON file
   - POST: If existing boundary selected, redirect to step 2 with boundary_id in session. If GeoJSON uploaded, parse it (validate it's valid GeoJSON with polygon/multipolygon), create Boundary record, redirect to step 2.

2. `setup_confirm(request)` — Step 2: Review boundary on map
   - GET: Load boundary from session, render map preview showing the boundary polygon, display area stats
   - POST: Confirm → redirect to step 3 (run)

3. `setup_run(request)` — Step 3: Execute auto_populate
   - GET: Show progress page with HTMX polling target
   - POST: Trigger auto_populate via call_command() in a synchronous execution (the command is fast enough for small boundaries). Store results in session.
   - Actually: use HTMX to POST to a run endpoint, which calls each auto_populate step sequentially and returns progress partial updates.

4. `setup_progress(request)` — HTMX endpoint for step-by-step progress
   - Called via hx-post, runs ONE step of auto_populate at a time
   - Tracks current step in session (basins → parcels → flowlines → stations)
   - Returns partial HTML showing completed steps with counts + current step spinner
   - When all steps done, returns final summary with link to dashboard

URL patterns (in setup/urls.py, app_name="setup"):
- path("", views.setup_wizard, name="wizard")
- path("confirm/", views.setup_confirm, name="confirm")
- path("run/", views.setup_run, name="run")
- path("progress/", views.setup_progress, name="progress")

Add to config/urls.py: path("setup/", include("setup.urls"))

For the auto_populate integration: Do NOT shell out to management command. Import and call the step methods directly from the Command class, or better, extract the step logic into a service layer function that both the command and the wizard can call. Create setup/services.py with:
- `run_auto_populate_step(boundary, step_name)` → returns (count, errors) tuple
- This calls the same logic as the management command steps but returns structured data instead of stdout printing.

Avoid: Don't use Celery or background tasks. The API calls for a single boundary are fast enough (5-30 seconds per step based on Phase 17/18 experience). The wizard runs steps one-at-a-time via HTMX polling.
