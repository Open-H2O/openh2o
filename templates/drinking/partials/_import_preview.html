{% comment %} The preview: what a commit WOULD do, before anything is written. The whole point of this screen is that an operator can see the consequences before accepting them, so it separates the two outcomes that are easy to confuse: • ERRORS block a row. It will not be created, and the reason is stated in the row's own language ("PS Code X is not a known sampling point"). • WARNINGS do not block anything. They are the two things an operator should know happened but that are not mistakes: an analyte being added to the vocabulary, and a result already on file being skipped. Deliberately no verdict language anywhere: this counts rows, it does not judge results. Caller supplies: recognised, preview_rows, error_rows, warning_rows, row_count, error_count, duplicate_count, new_analytes, committable, rows_json. {% endcomment %}
