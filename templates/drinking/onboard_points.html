Why this step exists

Every row of a lab file names a PS Code — the state's identifier for the exact place a sample was drawn, written as {{ system.pwsid }}_{facility}_{point}. The import matches on it and never invents one, so a system with facilities but no sampling points cannot take a lab file at all: every row would come back as an unknown PS Code.

You supply the last part — a point number and a name. The PWSID and the facility are already known, so the full code is composed for you and stored exactly as shown.

“Facility” here does not only mean a well. A system's distribution system is a facility too — usually DST — and the Lead & Copper and disinfection-byproduct points that sit out in the pipes hang off it, with point numbers like LCR or 900. Those are ordinary points on an ordinary facility; add them the same way.
