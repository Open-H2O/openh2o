What the import will and will not create

Sample events and results are created from the file. Analytes the file names but this system has never seen are added to the vocabulary, because the analyte list belongs to the regulator and the state's own file is the authority on it — the preview lists every one before you commit.

Sampling points and facilities are never created. A PS Code this system does not carry is reported as a row error, so a mistyped code cannot silently invent a monitoring location. Add the facility and sampling point first, then re-import.

Re-importing a file you have already imported is safe. Results matching one already recorded are flagged as duplicates in the preview and skipped on commit.
